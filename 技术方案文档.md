# 掌上移动PLC编程助手 - 技术方案文档

## 一、项目概述

### 1.1 项目定位
运行在微信内的移动端虚拟PLC编程与仿真工具，参考Siemens TIA Portal（博途）的梯形图编程方式。

### 1.2 核心价值
- 手机端完成梯形图编辑
- 虚拟PLC仿真运行
- 在线监控及辅助计算
- 无需连接真实硬件
- 不含通信和运动控制功能

---

## 二、技术架构

### 2.1 推荐技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | **uni-app** | 一套代码多端运行，优化微信小程序体验 |
| 语言 | TypeScript | 类型安全，便于维护复杂逻辑 |
| 梯形图渲染 | Canvas 2D | 高性能绑图，满足50ms响应要求 |
| 状态管理 | Pinia | 轻量级，适合微信小程序内存限制 |
| 本地存储 | Storage API | 微信Storage，持久化项目数据 |
| 仿真引擎 | TypeScript Worker | 与UI分离，避免阻塞主线程 |

### 2.2 性能指标

| 指标 | 目标值 | 实现方案 |
|------|--------|----------|
| 运行内存 | ≤300MB | Canvas离屏渲染、对象池复用 |
| 缩放响应 | <50ms | Canvas局部重绘、缩放锚点缓存 |
| 拖拽响应 | <50ms | requestAnimationFrame、脏矩形更新 |
| 编辑流畅度 | 50网络无卡顿 | 虚拟列表、视口裁剪 |

### 4.2 梯形图编辑器

#### 4.2.1 数据结构

```typescript
interface Network {
  id: string
  number: number
  elements: LadderElement[]
  connections: Connection[]
}

interface LadderElement {
  id: string
  type: ElementType  // NO, NC, COIL, TON, CTU, etc.
  address: string    // I0.0, Q0.1, M10.0
  symbol: string      // 符号名
  position: { x: number, y: number }
  size: { width: number, height: number }
  params: Record<string, any>  // PT, PV等参数
}

interface Connection {
  from: string  // elementId
  to: string    // elementId
  type: 'power' | 'branch'
}
```

#### 4.2.2 指令集实现

| 类别 | 指令 | 说明 |
|------|------|------|
| 位逻辑 | NO (常开) | 输入为TRUE时导通 |
| | NC (常闭) | 输入为FALSE时导通 |
| | Coil (线圈) | 写入输出 |
| | SR/RS | 置位优先/复位优先触发器 |
| | 边沿 | P/N 上升沿/下降沿检测 |
| 定时器 | TON | 通电延时 |
| | TOF | 断电延时 |
| | TP | 脉冲 |
| | TONR | 保持型通电延时 |
| 计数器 | CTU | 递增计数 |
| | CTD | 递减计数 |
| | CTUD | 递增递减计数 |
| 比较 | ==, !=, >, <, >=, <= | 支持Int/DInt/Real |
| 数学 | +, -, *, /, ABS, SQRT | 算术运算 |
| 移动 | MOVE, BLKMOVE | 数据移动 |
| 转换 | INT/DINT/REAL | 类型转换 |
| 字逻辑 | AND, OR, XOR | 位逻辑运算 |
| 移位 | SHL, SHR, ROL, ROR | 移位循环 |

### 4.3 变量表

```typescript
interface Variable {
  id: string
  symbol: string      // 符号名
  address: string     // I0.0, Q0.1, MB10, DB1.DBD0
  dataType: DataType  // Bool, Byte, Word, DWord, Int, DInt, Real, String
  comment: string
  array?: { length: number }
  struct?: StructDefinition
  retain: boolean     // 掉电保持
  initialValue?: any
}

enum DataArea {
  I = 'I',      // 输入
  Q = 'Q',     // 输出
  M = 'M',     // 位存储
  DB = 'DB'    // 数据块
}
```

### 4.4 虚拟PLC引擎

```typescript
class PLCEngine {
  // 内存区
  inputs: BitArray        // I区
  outputs: BitArray       // Q区
  memory: BitArray         // M区
  dataBlocks: Map<string, DataBlock>

  // 定时器/计数器状态
  timers: Map<string, TimerState>
  counters: Map<string, CounterState>

  // 扫描控制
  isRunning: boolean
  scanTime: number        // ms

  // 核心方法
  scan(): void            // 一次完整扫描
  executeNetwork(net: Network): boolean  // 执行单个网络
  executeElement(elem: LadderElement): any  // 执行单个指令

  // 监控
  getValue(address: string): any
  forceValue(address: string, value: any): void
}
```

## 七、关键技术难点

### 7.1 梯形图Canvas渲染
```
优化策略：
1. 分层渲染：背景层(静态) + 前景层(动态) + 交互层(临时)
2. 脏矩形更新：只重绘变化区域
3. 对象池：复用Canvas Path对象
4. 离屏预渲染：复杂元件预先生成
5. 视口裁剪：只渲染可见区域
```

### 7.2 触摸交互
```
优化策略：
1. 手势识别：双指缩放(ScaleGestureDetector) + 单指拖拽
2. 碰撞检测：四叉树优化大量元件查询
3. 吸附算法：网格吸附 + 元件连接点吸附
4. 橡皮筋效果：超出边界时的弹性反馈
```

### 7.3 仿真引擎性能
```
优化策略：
1. Web Worker：仿真逻辑独立线程
2. 增量执行：只执行变化的逻辑
3. 批量更新：合并UI刷新
4. 状态缓存：避免重复计算
```
